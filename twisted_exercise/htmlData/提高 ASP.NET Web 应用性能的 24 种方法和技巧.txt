<script type="text/javascript">
    $(function()
    {
        //取消背景层的滚动条滚动样式
        /*$(document).css(
            {"overflow":"none","overflow":"hidden","_overflow":"hidden","padding":"0 17px 0 0"}
        );*/

        $('#accept').removeAttr('checked');
        
        //申请创建主题弹框
        $('body').delegate('.apply_CreatF', 'click', function()
        {
            var login_username = '';
            var cookie = document.cookie.split(";");
            for (var i in cookie){
                var val = cookie[i];
				var tmp = val.split('=');
				if (tmp[0].trim() == 'UserName'){
					login_username = tmp[1];
				}
            }
        	if (login_username == ''){
				location.href = "https://passport.csdn.net/account/login";
            }
        	$(".mark_apply_create").appendTo('body').removeClass("hide").fadeIn(100);
        	$(".mark_apply_create").find(".popup_c:eq(0)").show().siblings().hide();

        });

        $("body").delegate('#accept', 'click', function()
        {
            checkState();
            document.documentElement.style.cssText = 'overflow:none;+overflow:hidden;_overflow:hidden;';
            document.body.style.cssText = 'overflow:hidden;+overflow:none;_overflow:none;padding:0 17px 0 0;';
        });

        //提交创建主题
        $('body').delegate('.create_forum_submit', 'click', function(){
            var tel = $(".popup_list input[type='tel']").val();
            if($('.input_forum_name').val().trim() == ""){
                $('.create_forum_error').html('请填写主题名称').show();
                $('.input_forum_name').focus();
            }else if($('.input_forum_describe').val().trim() == ""){
                $('.create_forum_error').html('请填写主题描述').show();
                $('.input_forum_describe').focus();
            //}else if(/^13\d{9}$/g.test(tel)||(/^15[8,9]\d{8}$/g.test(tel)||(/^15[8,9]\d{8}$/g.test(tel))
            }else if(/^1[3||5||8]\d{9}$/g.test(tel))
            {
                $(this).attr("disabled", "disabled");
                var data = $('.create_forum_form').serialize();
                $.post("/forum/create", data, function(result){
                    if(result.status){
                        $('.create_forum_error').hide();
                        $(".mark_apply_create").find(".popup_c:eq(2)").show().siblings().hide();
                    } else{
                        $('.create_forum_submit').attr("disabled", false);
                        $('.create_forum_error').html(result.error).show();
                    }
                });
            }
            else
            {
                $('.create_forum_error').html('请填写正确的手机号').show();
                $(".popup_list input[type='tel']").focus();
            }
			return false;
        });
        
        //状态检查
        function checkState()
        {
            var flag = false;
            var isCheck = $("#accept").prop("checked");
            if(isCheck != null && isCheck == true)
            {
                flag = true;

            }
            if(flag)
            {
            	$('#next').removeAttr("disabled").addClass("next_02");
                $('body').delegate('#next', 'click', function()
                {
                    $(".mark_apply_create").find(".popup_c:eq(1)").show().siblings().hide();

                });
            }
            else
            {
                $('#next').attr("disabled",true).removeClass('next_02');
            }
            //alert(flag);

        };


        $("body").delegate('.close', 'click', function()
        {
        	$(".mark_apply_create").addClass("hide");
            document.documentElement.style.cssText = '';
            document.body.style.cssText = '';
        });
        
    });
</script>

				<div class="news_description ">
                        <div class="description markdown_views">
                <p>在这篇文章中，将介绍一些提高 ASP.NET Web 应用性能的方法和技巧。众所周知，解决性能问题是一项繁琐的工作，当出现性能问题，每个人都会归咎于编写代码的开发人员。</p><p><strong>以下为译文</strong></p><p><img src="http://img.blog.csdn.net/20160122191748655" alt="图片描述" title=""></p><p>那性能问题到底该如何解决？以下是应用系统发布前，作为 .NET 开发人员需要检查的点。</p><p><strong>1.debug=「false」</strong></p><p>当创建 ASP.NET Web应用程序，默认设置为「true」。开发过程中，设置为「true」是非常有用多，但在应用程序发布部署时，需将其设置为「false」。</p><pre class="prettyprint"><code class=" hljs lua">&lt;compilation defaultLanguage=<span class="hljs-string">"C#"</span> <span class="hljs-built_in">debug</span>=<span class="hljs-string">"false"</span> targetFramework=<span class="hljs-string">"4.0"</span> /&gt;  </code></pre><p><strong>2.关闭 tracing(追踪)</strong></p><p>tracing 是非常可怕的，你有没有忘记关闭它。假如没用，请确定编辑 web.config 并且关闭它。它将占用大量您的程序资源。</p><pre class="prettyprint"><code class=" hljs xml"><span class="hljs-tag">&lt;<span class="hljs-title">trace</span> <span class="hljs-attribute">enabled</span>=<span class="hljs-value">"false"</span> <span class="hljs-attribute">requestLimit</span>=<span class="hljs-value">”10”</span> <span class="hljs-attribute">pageoutput</span>=<span class="hljs-value">”false”</span> <span class="hljs-attribute">traceMode</span>=<span class="hljs-value">”SortByTime”</span> <span class="hljs-attribute">localOnly</span>=<span class="hljs-value">”true”</span>&gt;</span>  </code></pre><p><strong>3.禁用 session</strong></p><p>假如您用不到 session 会话跟踪请务必禁用它。您可以在每个 asp.net 页面中设置如下:</p><pre class="prettyprint"><code class=" hljs mel">&lt;<span class="hljs-variable">%@</span> page language=<span class="hljs-string">"c#"</span> codebehind=<span class="hljs-string">"webform1.aspx.cs"</span> autoeventwireup=<span class="hljs-string">"false"</span> inherits=<span class="hljs-string">"webapplication1.webform1"</span>

enablesessionstate=<span class="hljs-string">"false"</span> <span class="hljs-variable">%&gt;</span>  </code></pre><p><strong>4.使用发布版本部署应用</strong></p><p>部署应用程序到生产环境时，要确保使用的发布版本模式，而不是调试模式。如果使用调试模板极容易发生请求超时。部署成发布版本，你将会发现速度有很大的提升。</p><p><strong>5.关闭页面的 View State</strong></p><p>View State 主要是在提交以后回显用的，它只有在页面中的数据是提交到本页时才有用。其默认是「true」。如果你没有使用表单数据回传，那么可以关闭 View State。</p><pre class="prettyprint"><code class=" hljs mel">&lt;<span class="hljs-variable">%@</span> Page EnableViewState=<span class="hljs-string">"false"</span> <span class="hljs-variable">%&gt;</span>  </code></pre><p><strong>6.避免使用 Response.Redirect</strong></p><p>Redirect(重定向)非常麻烦，它仅用于用于从当前物理服务器开发跳转到其它服务器。如果只是在本服务器开发内页面跳转请使用 Server.Transfer 语法，这样会减少很多没有必要的客户端重定向。</p><p><strong>7.运用 StringBuilder 类以及使用 ToString()方法</strong></p><p>String 类对象是不可改变的，对于 String 对象的重新赋值在本质上是重新创建了一个 String 对象并将新值赋予该对象，其方法 ToString 对性能的提高并非很显著。在处理字符串时，最好使用 StringBuilder 类，其 .NET 命名空间是 System.Text。该类并非创建新的对象，而是通过 Append，Remove，Insert 等方法直接对字符串进行操作，通过 ToString 方法返回操作结果。 其定义及操作语句如下所示</p><pre class="prettyprint"><code class=" hljs axapta"><span class="hljs-keyword">int</span> num;  
　　System.Text.StringBuilder <span class="hljs-keyword">str</span> = <span class="hljs-keyword">new</span> System.Text.StringBuilder(); <span class="hljs-comment">//创建字符串 </span>
　　<span class="hljs-keyword">str</span>.Append(num.ToString()); <span class="hljs-comment">//添加数值num </span>
　　Response.Write(<span class="hljs-keyword">str</span>.ToString); <span class="hljs-comment">//显示操作结果</span></code></pre><p><strong>8.避免抛出异常</strong></p><p>异常会导致速度变慢，并且使得应用程序页面显示异常，使得无法进行其他操作。可以使用 try / catch 使出现的异常记录到日志文件中。</p><p><strong>9.使用 finally 方法回收资源</strong></p><p>假如你在应用开发中大量使用其他数据库连接和访问文件，请确定在用完后关闭它们。finally 块是程序中最后被执行，因此在这里面的代码会确保一定会被执行，关闭代码一定要在这个开发方法块中执行。</p><p><strong>10.使用客户端脚本验证</strong></p><p>用客户端验证代替服务器开发端验证。服务器开发端数据验证将会大量消耗您的服务器开发上的资源，并且会代来大量的页面数据回传。</p><p><strong>11.使用 Page.IsPostback</strong></p><p>请确保不执行过多的回传代码。使用 Page.IsPostBack 属性，以确保只执行页面初始化逻辑，当一个页面第一次加载，而不向响应客户回发。</p><p><strong>12.使用分页</strong></p><p>大多数 Web 应用程序数据是以表格形式显示。分页有利用应用开发程序效率。每次尽量显示小部分数据，这样会加快页面显示速度。</p><p><strong>13.使用 Ajax 异步调用</strong></p><p>使用 Ajax 方法，进行异步调用。</p><p><strong>14.删除未使用的 HttpModules</strong></p><p>对于 httpModules，我们可以理解为：建立一个通用的可用被插入任何 Web 应用程序的 HttpApplication 事件钩子。使用 HttpModule 是可复用的，不需要特定语应用程序代码的，只需要 web.config 中的一个条目。 在 web.config 文件中，删除未使用的 HttpModules。</p><p><strong>15.避免递归函数/嵌套循环</strong></p><p>在任何编程语言中都需避免嵌套循环和递归函数，以提高性能。</p><p><strong>16.不使用不必要的 Server Control</strong></p><p>ASP.NET 中，大量的服务器端控件方便了程序开发，但也可能带来性能的损失，因为用户每操作一次服务器端控件，就产生一次与服务器端的往返过程。因此，非必要，应当少使用 Server Control。</p><p><strong>17.调用多个操作时，请使用多线程</strong></p><p>问题出现时，单线程卡在此问题上长时间运行。因此，可以使用多个线程以提高应用程序的响应速度。</p><p><strong>18.数据库的连接和关闭</strong></p><p>访问数据库资源需要创建连接、打开连接和关闭连接几个操作。这些过程需要多次与数据库交换信息以通过身份验证，比较耗费服务器资源。ASP.NET 中提供了连接池(Connection Pool)改善打开和关闭数据库对性能的影响。系统将用户的数据库连接放在连接池中，需要时取出，关闭时收回连接，等待下一次的连接请求。连接池的大小是有限的，如果在连接池达到最大限度后仍要求创建连接，必然大大影响性能。因此，在建立数据库连接后只有在真正需要操作时才打开连接，使用完毕后马上关闭，从而尽量减少数据库连接打开的时间，避免出现超出连接限制的情况。</p><p><strong>19.将 SqlDataReader 类用于快速只进数据游标</strong></p><p>SqlDataReader 类提供了一种读取从 SQL Server 数据库检索的只进数据流的方法。如果当创建 ASP.NET 应用程序时出现允许您使用它的情况，则 SqlDataReader 类提供比 DataSet 类更高的性能。情况之所以这样，是因为 SqlDataReader 使用 SQL Server 的本机网络数据传输格式从数据库连接直接读取数据。另外，SqlDataReader 类实现 IEnumerable 接口，该接口也允许您将数据绑定到服务器控件。有关更多信息，请参见 SqlDataReader 类。有关 ASP.NET 如何访问数据的信息，请参见通过 ASP.NET 访问数据。</p><p><strong>20.高性能的 SQL 语句规则</strong></p><ul>
<li>尽量避免全表扫描</li>
<li>尽量避免在 where 子句中对字段进行 null 值判断</li>
<li>尽量避免在 where 子句中使用 != 或 &lt;&gt; 操作符</li>
<li>尽量避免在 where 子句中使用 or 来连接条件</li>
<li>in 和 not in 也要慎用</li>
<li>不要在 where 子句中的「=」左边进行函数、算术运算或其他表达式运算</li>
<li>Update 语句，如果只更改1、2个字段，不要Update全部字段</li>
<li>对于多张大数据量（这里几百条就算大了）的表JOIN，要先分页再JOIN，否则逻辑读会很高，性能很差</li>
<li>尽可能的使用 varchar/nvarchar 代替 char/nchar 更多规则方法请参照：<a href="http://database.51cto.com/art/201407/445934.htm">http://database.51cto.com/art/201407/445934.htm</a></li>
</ul><p><strong>21.缓存</strong></p><p>缓存是一种用空间换取时间的技术，通俗点也就是说把你得到的数据存放在内存中一段时间，在这短时间内服务器不去读取数据库、或是真实的数据源，而是读取你存放在内存中的数据。 缓存是网站性能优化不可缺少的一种数据处理机制，他能有效的缓解数据库压力。 ASP.NET 中的缓存主要分为：</p><ul>
<li>页面缓存</li>
<li>数据源缓存</li>
<li>自定义数据缓存</li>
</ul><p><strong>22.做负载均衡和服务器加成</strong></p><p>负载均衡不应该仅仅被看作是实现可扩展性的一种手段。尽管它肯定提高了可扩展性，很多时候它增加了 Web 应用程序的性能，因为请求和用户都散发着多个服务器。</p><p>23.通过 FxCop 做代码检查和优化</p><p>FxCop 是一个代码分析工具，它使用基于规则的引擎，来检查出你代码中不合规范的部分；你也可以定制自己的规则加入到这个引擎。其中的一些规则是：</p><ul>
<li>避免过多的局部变量</li>
<li>避免使用未调用的私有代码</li>
<li>避免未实例化的内部类</li>
<li>避免使用未密封的特性</li>
<li>避免进行不必要的强制转换</li>
<li>以内联方式初始化引用类型的静态字段</li>
<li>用 NeutralResourcesLanguageAttribute 标记程序集</li>
<li>将成员标记为 Static 等等。</li>
</ul><p><strong>24.ASP.NET 性能监控工具</strong></p><p>这些是用于监视代码的性能的工具。</p><ul>
<li>.NET 内存分析器</li>
<li>Red Gate ANTS 性能分析工具</li>
<li>Fiddler</li>
<li>性能计数器</li>
</ul><p>结论： 以上是一些性能调整的提示。性能调优不是一天两天的工作，而是一个反反复复的过程。对于网站开发人员来说，在编写 ASP.NET 应用程序时注意性能问题，养成良好的习惯，提高应用程序性能，至少可以推迟必需的硬件升级，降低网站的成本。（文：<a href="http://www.oneapm.com/?utm_source=CSDN&amp;utm_medium=Article&amp;utm_term=%E6%8F%90%E9%AB%98%20ASP.NET%20Web%20%E5%BA%94%E7%94%A8%E6%80%A7%E8%83%BD%E7%9A%84%2024%20%E7%A7%8D%E6%96%B9%E6%B3%95%E5%92%8C%E6%8A%80%E5%B7%A7&amp;utm_campaign=AiNetArti&amp;from=jscldficktjscldfichp">OneAPM</a>工程师 责编/仲浩））</p>            </div>
